Index Design Rationale
idx_customer_state / idx_customer_city

Customers are frequently filtered by geographic location for analytics and reports. These indexes turn full table scans into efficient index lookups.

idx_home_policy_cust / idx_auto_policy_cust

Every customer dashboard and detail page joins policies to customers. These indexes dramatically speed up the JOIN operation on CUST_ID.

idx_home_policy_status / idx_auto_policy_status

Dashboard stats and policy listings frequently filter by active/expired status. With only 2 distinct values, the optimizer uses these efficiently for partial-table reads.

idx_home_invoice_policy / idx_auto_invoice_policy

Invoice pages join invoices to policies. These foreign key indexes optimize the JOIN and also speed up cascading deletes when a policy is removed.

idx_vehicle_policy

Vehicle listings are always filtered by auto policy. This index enables efficient lookups of vehicles belonging to a specific policy.

idx_user_role / idx_user_custid

Authentication and authorization queries filter users by role and customer ID on every request. These indexes keep login and session validation fast.
